## 模型配置

编辑项目根目录下的 `.env` 文件，配置 LLM 提供商。系统支持三种模式，按需选择一种即可。

### 方案一：DashScope 云端 API（推荐新手）

无需 GPU，注册阿里云 DashScope 即可使用。

**步骤：**

1. 访问 https://dashscope.console.aliyun.com/ 开通服务，获取 API Key
2. 编辑 `.env`：

```ini
LLM_PROVIDER=dashscope
DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
DASHSCOPE_MODEL=qwen-plus
```

可选模型：`qwen-plus`、`qwen-turbo`、`qwen-max`

### 方案二：OpenAI 兼容 API

适用于 OpenAI、DeepSeek、Moonshot 等任何兼容 OpenAI 接口的服务。

```ini
LLM_PROVIDER=openai
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini
```

DeepSeek 示例：

```ini
LLM_PROVIDER=openai
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
OPENAI_BASE_URL=https://api.deepseek.com/v1
OPENAI_MODEL=deepseek-chat
```

### 方案三：本地模型（vLLM）

需要 NVIDIA GPU（建议 16GB+ 显存）。使用 vLLM 部署模型，后端通过 OpenAI 兼容 API 调用。

**步骤：**

1. 安装 vLLM：

```bash
pip install vllm
```

2. 下载模型权重（以 DeepSeek-R1-7B 为例）：

```bash
# 使用 huggingface-cli 下载
pip install huggingface_hub
huggingface-cli download deepseek-ai/DeepSeek-R1-Distill-Qwen-7B --local-dir ./model/deepseek-r1-7b
```

3. 启动 vLLM 推理服务：

```bash
# 基础启动
vllm serve ./model/deepseek-r1-7b --port 8001

# 指定参数启动
vllm serve ./model/deepseek-r1-7b \
  --served-model-name deepseek-r1-7b \
  --host 0.0.0.0 \
  --port 8001 \
  --dtype auto \
  --max-model-len 4096
```

4. 验证服务是否正常：

```bash
curl http://localhost:8001/v1/models
```

5. 编辑 `.env`：

```ini
LLM_PROVIDER=local
LOCAL_MODEL_BASE_URL=http://localhost:8001/v1
LOCAL_MODEL_API_KEY=not-needed
PROFESSIONAL_MODEL_NAME=deepseek-r1-7b
LIVING_MODEL_NAME=qwen-chat-7b
MODEL_PATH=./model
```

> **说明**：`PROFESSIONAL_MODEL_NAME` 需与 `vllm serve --served-model-name` 一致。如需同时部署两个模型，可启动两个 vLLM 实例分别监听不同端口，或在同一个实例中加载多个模型。

## 启动后端

```bash
cd backend

# 安装依赖
pip install -e ".[dev]"

# 启动服务（默认 8000 端口）
uvicorn app.main:app --reload --port 8000
```

验证后端：

```bash

curl http://localhost:8000/health
```

## 启动前端

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

浏览器访问 http://localhost:3000

### 前端环境变量

如需连接非默认后端地址，创建 `frontend/.env.local`：

```ini
NEXT_PUBLIC_API_URL=http://localhost:8000
```

### 生产构建

```bash
cd frontend
npm run build
npm start
```

生产模式默认监听 3000 端口，可通过 `PORT` 环境变量修改。

## Docker 一键部署

适用于有 GPU 的服务器，自动启动 vLLM + 后端 + 前端三个容器。

**步骤：**

1. 准备模型权重：

```bash
mkdir -p model
huggingface-cli download deepseek-ai/DeepSeek-R1-Distill-Qwen-7B --local-dir ./model/deepseek-r1-7b
```

2. 配置环境变量：

```bash
cp .env.example .env
# 编辑 .env，确保以下配置正确：
# PROFESSIONAL_MODEL_NAME=deepseek-r1-7b
# MODEL_PATH=./model
```

3. 启动：

```bash
docker-compose up -d
```

4. 访问：

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost:3000 |
| 后端 API | http://localhost:8000 |
| vLLM 推理 | http://localhost:8001 |
| Swagger 文档 | http://localhost:8000/docs |

5. 查看日志：

```bash
# 所有服务
docker-compose logs -f

# 仅后端
docker-compose logs -f backend

# 仅 vLLM
docker-compose logs -f vllm
```

6. 停止：

```bash
docker-compose down
```

## RAG 索引构建

向量索引已预构建在 `rag/vector_store/` 目录下，通常无需重新构建。

如需重建（例如添加了新的 .docx 文档）：

```bash
cd rag

# 构建索引
python -m rag.build_index --build

# 完全重建（删除旧索引）
python -m rag.build_index --rebuild

# 修复乱码文件名
python -m rag.build_index --fix-names
```

> 需要设置 `DASHSCOPE_API_KEY` 环境变量（用于文本向量化）。

## 常见问题

### Q: 启动报错 `DASHSCOPE_API_KEY not set`

即使使用本地模型，RAG 的文本嵌入（embedding）仍依赖 DashScope。在 `.env` 中配置 `DASHSCOPE_API_KEY`。

### Q: vLLM 启动报 CUDA 内存不足

尝试减小 `--max-model-len` 参数：

```bash
vllm serve ./model/deepseek-r1-7b --max-model-len 2048
```

或使用量化模型：

```bash
vllm serve ./model/deepseek-r1-7b --quantization awq
```

### Q: 前端连不上后端

确认后端已启动且 CORS 配置正确。后端默认允许所有来源（`allow_origins=["*"]`）。

如果是 Docker 部署，前端通过 `http://backend:8000` 访问后端（容器内网络），浏览器通过 `http://localhost:8000` 访问。

### Q: 模型切换后没有生效

调用 `POST /models/switch` 切换模型后，新的 LLM 实例会替换旧实例。当前进行中的对话不会中断，新对话使用新模型。

### Q: 如何添加新的民族知识库

1. 在 `rag/datasets/` 下新建民族目录（如 `rag/datasets/苗族/`）
2. 放入 `.docx` 文件
3. 重建索引：`python -m rag.build_index --rebuild`
4. 重启后端服务
